# 設計ドキュメント

このディレクトリには、設計作業時に作成されたドキュメントを保存します。

## ファイル命名規則

CLAUDE.mdで定義されている通り、以下の命名規則に従ってください：

```
YYYYMMDD_HHMM_{作業内容}.md
```

### 例

- `20250815_1430_ユーザー認証システム設計.md`
- `20250820_1000_データベーススキーマ設計.md`
- `20250825_1500_API設計.md`

## ドキュメントの種類

- **機能設計**: 新機能の要件・仕様
- **アーキテクチャ設計**: システム構成・レイヤー設計
- **データベース設計**: スキーマ・マイグレーション計画
- **API設計**: エンドポイント・リクエスト/レスポンス仕様
- **リファクタリング計画**: 改善案・移行計画

## 利点

- **時系列で設計意図を追跡できる**
- **LLMが過去の設計を参照可能**
- **チーム共有が容易**
- **意思決定の記録**
